What is this?

A design tool for simulating and exploring game progression and choices.

It's an interactive hybrid of a State Machine and a Markov Chain.

Controls

  • Double click the graph: create new state node.
  • Double click an edge: create an edge route.
  • Alt + drag from node to node: create a directed edge between nodes.
  • Shift + Drag: Select area.
  • Hold Shift: allows you to activate any state node.
  • D: duplicate selection.
  • G: group selection.
  • A: toggle auto-progression for selected node.
  • F: focus viewport on activated node.
  • Del: delete selection.

How do scripts work?

A state node's script is evaluated when that node is activated. You can use them for keeping track of state changes, for example increasing the player's currency: g.money += 100. Variables will be shown either in the side panel or on the node, but you can hide specific variables by naming them with an underscore at the start: g._hiddenNumber = 42

The script is just Javascript, but you have some pre-defined variables for you to use:

  • g. a global namepsace that any state node can access.
  • self. a local namespace that only the activating node can access.
  • self._visits how many times this state has been visited.
  • self._id the ID of this node.
  • threadId thread id of "thread" running this script.

There are also some helper functions:

  • random() generates a random number from 0 up to 1.
  • min(a, b) picks the lowest of a or b.
  • max(a, b) picks the highest of a or b.
  • log(message) displays the message in the side bar for 5s.
  • runNode(idOrTitle, threadId) activates a state node, threadId can be an abitrary name you create to start a new thread, or an existing thread id.
  • runThread(threadId) advances a 'thread', as if you had pressed the 🎲 button.
  • stopThread(threadId) kills a 'thread'.
  • changeThreadId(threadId, newId) changes a 'thread' id to a new arbitrary string of your choice.

Note that threads aren't like actual OS threads, they're just a term I use to indicate one player's movement across the graph.

How do weights work?

Weights typically indicate the likelyhood of that edge being travelled across, when you press the 🎲 button on a node it will travel across a random connected edge but higher weight edges will be more likely to be picked.

You can disable travel across an edge entirely by setting the weight to zero.

How do edge scripts work?

Edge scripts are evaluated whenever a connected state node is activated. When run they update the weight of the edge.

The script either needs to evaluate to a number, i.e. 3 * 2 or evaluate to a true or false value, i.e. g.playerHasGun

Like state nodes, edge scripts have access to the global namespace g., but you also access to some more:

  • from. the state node this edge is going from.
  • to. the state node this edge is going to.
  • self. the edge this weight script is attached to.
